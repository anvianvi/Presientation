Development Practices and Tools:
DRY (Don't Repeat Yourself)
The DRY principle is stated as

"Every piece of knowledge must have a single, unambiguous, authoritative representation within a system"
  • easier maintenance
  • less bugs
  • better scalability
  • increases code readability
examples
						
							// Before Refactoring
							function calculatePettersMoney (days, rate) {
								return days * rate;
							}
							
							function calculateAngelsMoney(days, rate) {
								return days * rate;
							}
							
							// Usage
							const petersInc = calculatePettersMoney(5, 7);
							const angelaInc = calculateAngelsMoney(4, 8);
						
					
          
						// After Refactoring
						function calculateInc(days, rate) {
							return days * rate;
						}
						
						// Usage
						const petersInc = calculateInc(5, 10);
						const angelaInc = calculateInc(4, 6);
